ESP32S3 固件开发框架选择

决定了"用什么语言写代码"和"难度有多大"

简单度 ⬆
A
Arduino (C++)

类C语法，DHT22/MQTT/WiFi库一行安装，社区资源海量。适合 80% IoT 项目。

生态成熟度 ⬆
B
MicroPython

Python语法，上手最快。但ESP32S3上有些库不如C++成熟，内存有限。

你选哪个？

A

Arduino (C++) — 推荐

库生态最完整，DHT22 + PubSubClient MQTT + WiFi 三件套开箱即用，教程多，遇到问题容易搜到答案

B

MicroPython

Python写起来舒服，快速实验方便。但MQTT库(umqtt)功能较弱，调试手段有限

推荐 Arduino 的理由

  • 腾讯 IoT SDK 官方支持 C/Arduino
  • DHT22、WiFiManager 等库一键安装
  • OTA 远程升级支持成熟
  • 你遇到问题时，网上 90% 的 ESP32 教程都是 Arduino

Arduino 的代价

  • C++ 语法比 Python 繁琐一点
  • 需要安装 Arduino IDE 或 PlatformIO
